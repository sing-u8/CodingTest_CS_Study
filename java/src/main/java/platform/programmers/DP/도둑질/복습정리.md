# 프로그래머스 Level 4 — 도둑질 (42897) 복습 노트

## 1. 문제 요약

마을의 집들이 **원형(Circle)**으로 배치되어 있고, 각 집에는 일정량의 돈이 있다. 도둑은 **인접한 두 집을 동시에 털면 경보가 울리므로**, 인접하지 않는 집들만 골라 털어야 한다. 각 집에 있는 돈이 배열 `money`로 주어질 때, 도둑이 훔칠 수 있는 **돈의 최댓값**을 구하라.

```
집 배치 (원형):
    [1]---[2]
     |     |
    [1]---[3]

→ 첫 번째 집과 마지막 집도 "인접"한 관계!
```

**제한사항**: 집의 개수는 3 이상 1,000,000 이하 → O(N) 풀이가 필요하다.

---

## 2. 기초 개념 — 표준 House Robber DP (선형 버전)

원형 문제를 풀기 전에, 먼저 **집들이 일직선으로 늘어서 있는 기본형**을 이해해야 한다. 이것이 "표준 House Robber DP"이다. (LeetCode 198번과 동일)

### 2.1 문제 설정

일직선으로 늘어선 집들이 있고, 인접한 두 집을 연속으로 털 수 없다. 이 제약 하에서 훔칠 수 있는 돈의 최댓값을 구해야 한다.

```
집:    [0]---[1]---[2]---[3]---[4]
돈:     2     7     9     3     1

인접한 집을 동시에 털 수 없다!
→ [0]과 [1] 동시 불가, [1]과 [2] 동시 불가, ...
→ 하지만 [0]과 [2]는 가능 (한 칸 건너뛰었으므로)
```

### 2.2 DP적 사고: "각 집 앞에서의 선택"

도둑이 i번째 집 앞에 서 있을 때, 선택지는 딱 **두 가지**뿐이다.

**선택 1 — "이 집을 턴다"**: 바로 전 집(i-1)은 경보 때문에 못 털었어야 한다. 따라서 `dp[i-2] + money[i]`, 즉 두 칸 전까지의 최적해에 현재 집의 돈을 더한다.

**선택 2 — "이 집을 안 턴다"**: 아무 제약이 추가되지 않으므로, `dp[i-1]`, 즉 직전 집까지의 최적해를 그대로 물려받는다.

이 두 선택 중 더 큰 쪽을 고르면 i번째 집까지 고려했을 때의 최적해가 된다.

### 2.3 점화식

```
dp[i] = max(dp[i-1], dp[i-2] + money[i])
              ↑              ↑
         안 터는 경우    터는 경우
```

### 2.4 왜 이 점화식이 "모든 경우"를 커버하는가?

"겨우 두 가지 선택만 고려하는데, 정말 모든 조합을 다 따지는 건가?"라는 의문이 들 수 있다.

핵심은 **dp[i]가 "0번 집부터 i번 집까지의 범위에서 가능한 모든 유효 조합의 최댓값"을 이미 내포하고 있다**는 점이다. dp[i-1]과 dp[i-2]가 각각 자기 범위 내의 최적해를 이미 보장하고 있으므로, i번째 집에서의 두 가지 선택만 비교하면 dp[i]도 자연스럽게 최적해가 된다. 이것이 DP의 **최적 부분 구조(Optimal Substructure)** 원리이다.

비유하자면, 계단을 한 층씩 올라가면서 "지금까지의 최고 기록"을 갱신해 나가는 것과 같다. 각 층에서 새로운 정보(현재 집의 돈)만 반영하면 되고, 과거의 모든 조합을 다시 탐색할 필요가 없다.

### 2.5 선형 House Robber 트레이스 예시

`money = [2, 7, 9, 3, 1]`에 대해 DP를 추적해 보자.

```
초기 조건:
  dp[0] = 2          ← 집이 하나뿐이면 그냥 턴다
  dp[1] = max(2, 7) = 7  ← 집 두 채 중 더 큰 걸 턴다

i=2: money[2]=9
  안 턴다 → dp[1] = 7
  턴다   → dp[0] + 9 = 2 + 9 = 11
  dp[2] = max(7, 11) = 11  ✓ (집 0과 집 2를 턺)

i=3: money[3]=3
  안 턴다 → dp[2] = 11
  턴다   → dp[1] + 3 = 7 + 3 = 10
  dp[3] = max(11, 10) = 11  ✓ (집 3을 터는 것보다 안 터는 게 나음)

i=4: money[4]=1
  안 턴다 → dp[3] = 11
  턴다   → dp[2] + 1 = 11 + 1 = 12
  dp[4] = max(11, 12) = 12  ✓ (집 0, 2, 4를 턺)

최종 답: dp[4] = 12
```

## 3. 공간 최적화: DP 배열 → 변수 두 개

### 3.1 관찰

점화식 `dp[i] = max(dp[i-1], dp[i-2] + money[i])`를 보면, dp[i]를 계산할 때 필요한 것은 **직전 값(dp[i-1])과 그 전 값(dp[i-2])** 딱 두 개뿐이다. 따라서 배열 전체를 유지할 필요 없이 변수 두 개로 충분하다.

```
prev1 → dp[i-1] 역할 (직전 집까지의 최적해)
prev2 → dp[i-2] 역할 (두 칸 전 집까지의 최적해)
```

### 3.2 코드 패턴

```java
int prev2 = 0;  // dp[i-2] 역할
int prev1 = 0;  // dp[i-1] 역할

for (int i = 0; i < n; i++) {
    int curr = Math.max(prev1, prev2 + money[i]);  // ① 새 최적해 계산
    prev2 = prev1;   // ② 밀어내기
    prev1 = curr;     // ③ 갱신
}
// prev1이 최종 답
```

### 3.3 컨베이어 벨트 비유

prev2와 prev1을 **컨베이어 벨트** 위의 슬롯 두 개라고 생각하면 직관적이다.

```
매 반복마다 벨트가 한 칸씩 이동한다:

반복 전:  [ prev2 ] [ prev1 ]  ← 이 두 값을 보고 새 값(curr)을 계산
반복 후:            [ prev2 ] [ prev1 ]
                       ↑         ↑
                    옛 prev1   새로 계산한 curr
```

### 3.4 상세 트레이스 (값의 변화 시점에 주의)

**중요**: prev2 값은 **계산에 사용되는 시점(계산 전)**과 **업데이트된 후(계산 후)**가 다르다. 아래 트레이스는 혼동이 없도록 "계산 전 상태"와 "계산 후 상태"를 명시한다.

`money = [1, 2, 3]` (Case A 예시)에 대한 트레이스:

```
시작 상태: prev2 = 0, prev1 = 0

i=0, money[0] = 1
┌─────────────────────────────────────────────────┐
│ 계산 전 상태:  prev2 = 0,  prev1 = 0            │
│                                                  │
│ ① curr = max(prev1, prev2 + money[0])           │
│        = max(  0,      0   +   1   )             │
│        = max(0, 1) = 1                           │
│                                                  │
│ ② prev2 = prev1 = 0   (이전 prev1을 보관)       │
│ ③ prev1 = curr   = 1   (새 최적해로 갱신)       │
│                                                  │
│ 계산 후 상태:  prev2 = 0,  prev1 = 1             │
│ 해석: "집 0까지 봤을 때 최대 = 1"               │
└─────────────────────────────────────────────────┘

i=1, money[1] = 2
┌─────────────────────────────────────────────────┐
│ 계산 전 상태:  prev2 = 0,  prev1 = 1            │
│                                                  │
│ ① curr = max(prev1, prev2 + money[1])           │
│        = max(  1,      0   +   2   )             │
│        = max(1, 2) = 2                           │
│                                                  │
│ ② prev2 = prev1 = 1                             │
│ ③ prev1 = curr   = 2                            │
│                                                  │
│ 계산 후 상태:  prev2 = 1,  prev1 = 2             │
│ 해석: "집 1까지 봤을 때 최대 = 2 (집 1만 턺)"   │
└─────────────────────────────────────────────────┘

i=2, money[2] = 3
┌─────────────────────────────────────────────────┐
│ 계산 전 상태:  prev2 = 1,  prev1 = 2            │
│                                                  │
│ ① curr = max(prev1, prev2 + money[2])           │
│        = max(  2,      1   +   3   )             │
│        = max(2, 4) = 4                           │
│                                                  │
│ ② prev2 = prev1 = 2                             │
│ ③ prev1 = curr   = 4                            │
│                                                  │
│ 계산 후 상태:  prev2 = 2,  prev1 = 4             │
│ 해석: "집 2까지 봤을 때 최대 = 4 (집 0+집 2)"   │
└─────────────────────────────────────────────────┘

최종 답: prev1 = 4
```

## 4. 핵심 풀이 — 원형 → 선형 두 번으로 분해

### 4.1 원형의 어려움

원형 구조에서는 **첫 번째 집(index 0)과 마지막 집(index n-1)이 인접**하다. 따라서 이 둘을 동시에 선택할 수 없다. 이 제약 때문에 선형 House Robber를 바로 적용할 수 없다.

### 4.2 핵심 아이디어: 두 개의 선형 문제로 분해

**"첫 번째 집을 턴다" vs "첫 번째 집을 안 턴다"** — 이 두 경우는 서로 배타적이므로, 각각을 독립적인 선형 문제로 풀고 둘 중 큰 값을 취한다.

**Case A**: 첫 번째 집을 포함할 수 있는 경우 → 마지막 집은 반드시 제외 → `money[0 .. n-2]` 범위에서 선형 DP

**Case B**: 마지막 집을 포함할 수 있는 경우 → 첫 번째 집은 반드시 제외 → `money[1 .. n-1]` 범위에서 선형 DP

**최종 답**: `max(Case A, Case B)`

### 4.3 왜 이 분해가 정확한가? (증명)

최적해에서 도둑이 실제로 선택한 집들의 집합을 S라 하자. 원형 배열에서 집 0과 집 n-1은 인접하므로, **S는 집 0과 집 n-1을 동시에 포함할 수 없다.** 따라서 모든 유효한 S는 반드시 다음 두 카테고리 중 하나에 속한다.

- S에 집 n-1이 없다 → S는 `[0, n-2]` 범위의 선형 문제의 해이기도 하다 (Case A에 포함)
- S에 집 0이 없다 → S는 `[1, n-1]` 범위의 선형 문제의 해이기도 하다 (Case B에 포함)

따라서 Case A와 Case B의 합집합이 모든 유효한 해를 빠짐없이 커버하며, 두 경우의 최댓값이 곧 전체 최적해가 된다.

### 4.4 원형 트레이스 예시

`money = [1, 2, 3, 1]` (집 4채, 원형)인 경우:

```
집 배치 (원형):
    [1]---[2]
     |     |
    [1]---[3]
```

**Case A**: `money[0..2] = [1, 2, 3]` (첫 번째 포함, 마지막 제외)

```
시작: prev2=0, prev1=0

i=0 │ money=1 │ 계산 전: prev2=0, prev1=0 │ curr=max(0, 0+1)=1 │ 계산 후: prev2=0, prev1=1
i=1 │ money=2 │ 계산 전: prev2=0, prev1=1 │ curr=max(1, 0+2)=2 │ 계산 후: prev2=1, prev1=2
i=2 │ money=3 │ 계산 전: prev2=1, prev1=2 │ curr=max(2, 1+3)=4 │ 계산 후: prev2=2, prev1=4

→ Case A 결과: 4  (집 0과 집 2를 턺: 1+3=4)
```

**Case B**: `money[1..3] = [2, 3, 1]` (첫 번째 제외, 마지막 포함)

```
시작: prev2=0, prev1=0

i=0 │ money=2 │ 계산 전: prev2=0, prev1=0 │ curr=max(0, 0+2)=2 │ 계산 후: prev2=0, prev1=2
i=1 │ money=3 │ 계산 전: prev2=0, prev1=2 │ curr=max(2, 0+3)=3 │ 계산 후: prev2=2, prev1=3
i=2 │ money=1 │ 계산 전: prev2=2, prev1=3 │ curr=max(3, 2+1)=3 │ 계산 후: prev2=3, prev1=3

→ Case B 결과: 3  (집 1만 턺: 3)
```

**최종 답**: `max(4, 3) = 4`

## 5. 솔루션 코드 (4개 언어)

### 5.1 JavaScript

```javascript
function solution(money) {
    const n = money.length;

    // 선형 House Robber: money[start..end] 범위에서 최대 합
    function rob(start, end) {
        let prev2 = 0; // dp[i-2]
        let prev1 = 0; // dp[i-1]
        for (let i = start; i <= end; i++) {
            const curr = Math.max(prev1, prev2 + money[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }

    // Case A: 첫 번째 집 포함 가능 (마지막 집 제외) → [0, n-2]
    // Case B: 마지막 집 포함 가능 (첫 번째 집 제외) → [1, n-1]
    return Math.max(rob(0, n - 2), rob(1, n - 1));
}
```

### 5.2 C++

```cpp
#include <vector>
#include <algorithm>
using namespace std;

int solution(vector<int> money) {
    int n = money.size();

    // 선형 House Robber: money[start..end] 범위에서 최대 합
    auto rob = [&](int start, int end) -> int {
        int prev2 = 0, prev1 = 0;
        for (int i = start; i <= end; i++) {
            int curr = max(prev1, prev2 + money[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    };

    // 원형 → 두 개의 선형 문제로 분해
    return max(rob(0, n - 2), rob(1, n - 1));
}
```

### 5.3 Java

```java
class Solution {
    public int solution(int[] money) {
        int n = money.length;
        // 원형 → 두 개의 선형 문제로 분해 후 최댓값
        return Math.max(rob(money, 0, n - 2), rob(money, 1, n - 1));
    }

    // 선형 House Robber: money[start..end] 범위에서 최대 합
    private int rob(int[] money, int start, int end) {
        int prev2 = 0, prev1 = 0;
        for (int i = start; i <= end; i++) {
            int curr = Math.max(prev1, prev2 + money[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}
```

### 5.4 Rust

```rust
fn solution(money: Vec<i32>) -> i32 {
    let n = money.len();

    // 선형 House Robber: money[start..=end] 범위에서 최대 합
    let rob = |start: usize, end: usize| -> i32 {
        let (mut prev2, mut prev1) = (0i32, 0i32);
        for i in start..=end {
            let curr = prev1.max(prev2 + money[i]);
            prev2 = prev1;
            prev1 = curr;
        }
        prev1
    };

    // 원형 → 두 개의 선형 문제로 분해
    rob(0, n - 2).max(rob(1, n - 1))
}
```

## 6. 복잡도 분석

**시간 복잡도: O(N)** — 선형 DP를 두 번 수행하므로 정확히 2N번 순회한다. N이 최대 1,000,000이므로 충분히 빠르다.

**공간 복잡도: O(1)** — DP 배열을 사용하지 않고 변수 두 개(prev2, prev1)만으로 이전 상태를 추적한다.

```
│     답 = max(Case A, Case B)                                 │
│                                                              │
│  4. 정당성: 최적해 S는 집 0과 집 n-1을 동시에 포함 불가     │
│     → 반드시 Case A 또는 Case B에 속함                       │
│     → 두 경우의 max가 전체 최적해                            │
│                                                              │
│  5. 복잡도: 시간 O(N), 공간 O(1)                            │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 8. 대안 풀이 — DP 배열 명시 버전 (학습용)

공간 최적화 버전이 가장 효율적이지만, DP의 구조를 더 명확하게 보고 싶다면 배열을 직접 사용하는 버전도 학습에 도움이 된다.

```java
class Solution {
    public int solution(int[] money) {
        int n = money.length;

        // dp1: 첫 번째 집 포함 가능 (0 ~ n-2)
        int[] dp1 = new int[n];
        dp1[0] = money[0];
        dp1[1] = Math.max(money[0], money[1]);
        for (int i = 2; i < n - 1; i++) {
            dp1[i] = Math.max(dp1[i - 1], dp1[i - 2] + money[i]);
        }

        // dp2: 마지막 집 포함 가능 (1 ~ n-1)
        int[] dp2 = new int[n];
        dp2[1] = money[1];
        dp2[2] = Math.max(money[1], money[2]);
        for (int i = 3; i < n; i++) {
            dp2[i] = Math.max(dp2[i - 1], dp2[i - 2] + money[i]);
        }

        return Math.max(dp1[n - 2], dp2[n - 1]);
    }
}
```

이 방식은 공간 O(N)을 사용하지만, dp[i]의 값이 어떻게 쌓이는지 디버깅이나 학습 목적으로 직접 확인할 수 있다는 장점이 있다.
