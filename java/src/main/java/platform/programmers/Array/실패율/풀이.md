## 1. 문제 요약

### 정의

**실패율** = 스테이지에 도달했으나 아직 클리어하지 못한 플레이어 수 ÷ 스테이지에 도달한 플레이어 수

### 입력

- `N`: 전체 스테이지 개수 (1 ≤ N ≤ 500)
- `stages`: 각 사용자가 현재 멈춰있는 스테이지 번호 배열 (길이 M, 1 ≤ M ≤ 200,000)
    - 값 범위: 1 이상 N+1 이하 (N+1은 모든 스테이지를 클리어한 경우)

### 출력

실패율이 높은 스테이지부터 내림차순으로 스테이지 번호를 담은 배열. 실패율이 같으면 작은 번호가 먼저 온다.

### 제약 조건에서 읽어내는 복잡도 상한

N ≤ 500, M ≤ 200,000이므로 O(N × M) ≈ 10^8도 허용 범위지만, 카운팅 배열을 쓰면 O(N + M)으로 훨씬 효율적이다.

## 2. 풀이 전략

### 핵심 아이디어: 카운팅 + 누적 차감

"스테이지 i에 도달한 사람 수"를 매번 stages 배열 전체를 순회해서 세면 O(N × M)이다. 하지만 다음 관찰을 이용하면 O(1)로 구할 수 있다.

> **관찰**: 스테이지 i에 도달한 사람 수 = 전체 플레이어 수 − (스테이지 1~(i-1)에서 멈춘 사람 수의 합)

따라서 `reached`라는 변수를 전체 플레이어 수로 초기화한 뒤, 스테이지 1부터 순서대로 처리하면서 **해당 스테이지에서 멈춘 사람 수를 빼나가면** 된다. 이것은 **컨베이어 벨트에서 각 정거장마다 사람이 내리는 것**과 같은 원리다. 전체 탑승객 수에서 앞 정거장에서 내린 수를 빼면 다음 정거장의 탑승객 수가 자동으로 나온다.

### 정렬 기준

1순위: 실패율 내림차순 (높은 실패율이 먼저)  
2순위: 스테이지 번호 오름차순 (같은 실패율이면 작은 번호가 먼저)

## 3. 내 풀이 (Java · HashMap 활용)

```java
import java.util.HashMap;

class Solution {
    public int[] solution(int N, int[] stages) {

        // [1단계] 카운팅 배열: 각 스테이지에 멈춰있는 플레이어 수 집계
        int[] challenger = new int[N + 2];
        for (int stage : stages) {
            challenger[stage] += 1;
        }

        // [2단계] 실패율 계산: HashMap에 (스테이지 번호 → 실패율) 저장
        HashMap<Integer, Double> fails = new HashMap<>();
        double total = stages.length; // 도달한 누적 인원 (double로 선언하여 자동 실수 나눗셈)

        for (int i = 1; i <= N; i++) {
            if (challenger[i] == 0) {
                // 해당 스테이지에 멈춘 사람이 없으면 실패율 0
                fails.put(i, 0.);
            } else {
                // 실패율 = 멈춘 수 / 도달한 수
                fails.put(i, challenger[i] / total);
                // 다음 스테이지 도달 인원 차감
                total -= challenger[i];
            }
        }

        // [3단계] Stream 정렬: 실패율 내림차순, 동률 시 번호 오름차순
        return fails.entrySet().stream().sorted((o1, o2) ->
            o1.getValue().equals(o2.getValue())
                ? Integer.compare(o1.getKey(), o2.getKey())
                : Double.compare(o2.getValue(), o1.getValue()))
            .mapToInt(HashMap.Entry::getKey).toArray();
    }
}
```

## 4. 내 풀이 상세 해설

### 4.1 — 카운팅 배열 (`challenger`)

`challenger[i]`는 현재 스테이지 i에 멈춰있는 플레이어 수이다. 배열 크기를 `N + 2`로 잡은 이유는 `stages` 배열에 `N + 1` 값(모든 스테이지를 클리어한 사람)이 들어올 수 있기 때문이다. `N + 1`로 잡으면 인덱스 `N + 1`에 접근할 때 ArrayIndexOutOfBoundsException이 발생한다.

### 4.2 — `total`을 `double`로 선언한 이유

`total`을 `int`가 아닌 `double`로 선언한 것이 핵심 포인트다. `challenger[i]`는 `int`인데, Java에서 `int / int`는 **정수 나눗셈**이 되어 소수점 이하가 버려진다. 하지만 `int / double`은 자동으로 **실수 나눗셈**으로 승격(promotion)되므로, 별도의 캐스팅 없이 정확한 실패율을 구할 수 있다. 배열 풀이에서 `(double) count[i] / reached`로 명시적 캐스팅하는 것과 같은 효과다.

### 4.3 — `challenger[i] == 0`일 때 `total`을 차감하지 않는 이유

`else` 블록에서만 `total -= challenger[i]`를 수행하는데, `if` 블록(challenger[i] == 0)에서 차감하지 않아도 문제가 없다. `challenger[i]`가 0이면 빼든 안 빼든 total 값이 변하지 않기 때문이다. 이것은 **의도적인 최적화**가 아니라 **논리적으로 불필요한 연산을 생략**한 것이다.

한 가지 주의할 점은, 만약 `total`이 0인 상태에서 `challenger[i]`도 0이면 `0 / 0.0`이 될 수 있다는 것인데, 이 코드에서는 `challenger[i] == 0` 조건을 먼저 검사하므로 이 경우에는 `if` 블록으로 빠져 나눗셈 자체가 실행되지 않는다. 다만 `total == 0`이고 `challenger[i] > 0`인 상황은 논리적으로 발생할 수 없다(도달한 사람이 0명인데 그 스테이지에 멈춘 사람이 있을 수 없으므로).

### 4.4 — Stream 정렬의 동작 원리

```java
fails.entrySet().stream().sorted((o1, o2) ->
    o1.getValue().equals(o2.getValue())
        ? Integer.compare(o1.getKey(), o2.getKey())
        : Double.compare(o2.getValue(), o1.getValue()))
    .mapToInt(HashMap.Entry::getKey).toArray();
```

이 코드의 흐름은 다음과 같다.

1. `fails.entrySet().stream()` — HashMap의 모든 (키, 값) 쌍을 스트림으로 변환
2. `sorted(...)` — 커스텀 비교 함수로 정렬
    - 두 엔트리의 실패율(value)이 **같으면** (`equals`) → 스테이지 번호(key) **오름차순** 비교
    - 실패율이 **다르면** → 실패율 **내림차순** 비교 (`o2`가 먼저)
3. `mapToInt(HashMap.Entry::getKey)` — 정렬된 엔트리에서 스테이지 번호(key)만 추출
4. `.toArray()` — int 배열로 변환

여기서 `equals()`로 비교하는 것이 `==`보다 안전하다. Double 객체끼리의 비교에서 `==`는 참조 비교가 될 수 있지만, `equals()`는 값 비교를 보장한다. (단, NaN 처리에서 미세한 차이가 있지만 이 문제에서는 NaN이 발생하지 않으므로 문제없다.)

### 4.5 — HashMap vs 배열 풀이의 차이

| 관점 | 내 풀이 (HashMap) | 배열 풀이 |
|---|---|---|
| 자료구조 | `HashMap<Integer, Double>` | `double[][]` 또는 별도 배열 |
| 실수 나눗셈 | `total`을 double로 선언하여 자동 승격 | 명시적 `(double)` 캐스팅 |
| 정렬 | Stream API + 람다 | `Arrays.sort` + 비교 함수 |
| 코드 스타일 | 함수형/선언적 | 절차적 |
| 공간 복잡도 | O(N) — HashMap 오버헤드 약간 더 큼 | O(N) — 배열이라 상수 계수 작음 |
| 시간 복잡도 | 동일: O(M + N log N) | 동일: O(M + N log N) |

두 풀이 모두 정확히 같은 알고리즘(카운팅 + 누적 차감 + 정렬)을 사용하며, 차이는 자료구조와 API 선택에 있다. HashMap 풀이는 Java의 Stream API를 활용한 간결한 표현이 장점이고, 배열 풀이는 메모리 효율과 직관적 인덱싱이 장점이다.

## 5. 트레이스 예시

### 입력: N=5, stages=[2, 1, 2, 6, 2, 4, 3, 3]

### 5.1 — 카운팅 단계

stages 배열을 순회하며 challenger 배열을 채운다.

| 인덱스 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
| challenger | 1 | 3 | 2 | 1 | 0 | 1 |

### 5.2 — 실패율 계산 단계 (누적 차감)

`total = 8.0` (전체 플레이어 수)으로 시작한다.

| 스테이지 i | challenger[i] | total (도달) | 실패율 계산 | 실패율 | total 차감 후 |
|---|---|---|---|---|---|
| 1 | 1 | 8.0 | 1 / 8.0 | 0.125 | 7.0 |
| 2 | 3 | 7.0 | 3 / 7.0 | ≈ 0.4286 | 4.0 |
| 3 | 2 | 4.0 | 2 / 4.0 | 0.5 | 2.0 |
| 4 | 1 | 2.0 | 1 / 2.0 | 0.5 | 1.0 |
| 5 | 0 | 1.0 | (조건: challenger==0) | 0.0 | 1.0 (차감 안 함) |

### 5.3 — 정렬 단계

정렬 전: {1→0.125, 2→0.4286, 3→0.5, 4→0.5, 5→0.0}

실패율 내림차순 정렬 (동률 시 번호 오름차순):

1. 스테이지 3 (0.5) — 4와 동률이지만 번호가 작으므로 먼저
2. 스테이지 4 (0.5)
3. 스테이지 2 (≈ 0.4286)
4. 스테이지 1 (0.125)
5. 스테이지 5 (0.0)

**최종 결과: [3, 4, 2, 1, 5]** ✓

## 6. 5개 언어 솔루션

### JavaScript

```javascript
function solution(N, stages) {
    // 각 스테이지에 멈춰있는 플레이어 수를 카운팅
    const count = new Array(N + 2).fill(0);
    for (const s of stages) {
        count[s]++;
    }

    // 실패율 계산: (스테이지 번호, 실패율) 쌍 생성
    const failRates = [];
    let reached = stages.length; // 현재 스테이지에 도달한 누적 인원

    for (let i = 1; i <= N; i++) {
        if (reached === 0) {
            // 도달한 사람이 없으면 실패율은 0
            failRates.push([i, 0]);
        } else {
            // 실패율 = 해당 스테이지에 멈춘 수 / 도달한 수
            failRates.push([i, count[i] / reached]);
        }
        // 다음 스테이지 도달 인원 = 현재 도달 - 현재 스테이지에서 멈춘 수
        reached -= count[i];
    }

    // 실패율 내림차순 정렬 (동률 시 스테이지 번호 오름차순)
    failRates.sort((a, b) => b[1] - a[1] || a[0] - b[0]);

    // 스테이지 번호만 추출하여 반환
    return failRates.map(([stage, _]) => stage);
}
```

### C++

```cpp
#include <vector>
#include <algorithm>
using namespace std;

vector<int> solution(int N, vector<int> stages) {
    // 각 스테이지에 멈춰있는 플레이어 수를 카운팅
    vector<int> count(N + 2, 0);
    for (int s : stages) {
        count[s]++;
    }

    // 실패율 계산: (실패율, 스테이지 번호) 쌍 생성
    vector<pair<double, int>> failRates;
    int reached = stages.size(); // 현재 스테이지에 도달한 누적 인원

    for (int i = 1; i <= N; i++) {
        double rate = 0.0;
        if (reached > 0) {
            // 실패율 = 해당 스테이지에 멈춘 수 / 도달한 수
            rate = (double)count[i] / reached;
        }
        failRates.push_back({rate, i});
        // 다음 스테이지 도달 인원 차감
        reached -= count[i];
    }

    // 실패율 내림차순, 동률 시 스테이지 번호 오름차순
    sort(failRates.begin(), failRates.end(), [](const auto& a, const auto& b) {
        if (a.first != b.first) return a.first > b.first;
        return a.second < b.second;
    });

    // 스테이지 번호만 추출
    vector<int> result;
    for (auto& [rate, stage] : failRates) {
        result.push_back(stage);
    }
    return result;
}
```

### Java (배열 기반 풀이)

```java
import java.util.*;

class Solution {
    public int[] solution(int N, int[] stages) {
        // 각 스테이지에 멈춰있는 플레이어 수를 카운팅
        int[] count = new int[N + 2];
        for (int s : stages) {
            count[s]++;
        }

        // 실패율 계산: (스테이지 번호, 실패율) 쌍 생성
        double[][] failRates = new double[N][2];
        int reached = stages.length; // 현재 스테이지에 도달한 누적 인원

        for (int i = 1; i <= N; i++) {
            failRates[i - 1][0] = i; // 스테이지 번호
            if (reached > 0) {
                // 실패율 = 해당 스테이지에 멈춘 수 / 도달한 수
                failRates[i - 1][1] = (double) count[i] / reached;
            } else {
                // 도달한 사람이 없으면 실패율은 0
                failRates[i - 1][1] = 0;
            }
            // 다음 스테이지 도달 인원 차감
            reached -= count[i];
        }

        // --- 정렬 방법 1 ---
        // 실패율 내림차순 정렬 (동률 시 스테이지 번호 오름차순)
        Arrays.sort(failRates, (a, b) -> {
            if (Double.compare(b[1], a[1]) != 0) return Double.compare(b[1], a[1]);
            return Double.compare(a[0], b[0]);
        });

        // 스테이지 번호만 추출하여 반환
        int[] result = new int[N];
        for (int i = 0; i < N; i++) {
            result[i] = (int) failRates[i][0];
        }
        return result;

        // --- 정렬 방법 2 ---
        /*
        return Arrays.stream(failRates)
            .sorted(Comparator
                   .comparingDouble((double[] a) -> a[1]).reversed()
                   .thenComparingDouble(a -> a[0]))
            .mapToInt(a -> (int) a[0])
            .toArray();
        */
    }
}
```

### Rust

```rust
fn solution(n: usize, stages: Vec<usize>) -> Vec<usize> {
    // 각 스테이지에 멈춰있는 플레이어 수를 카운팅
    let mut count = vec![0usize; n + 2];
    for &s in &stages {
        count[s] += 1;
    }

    // 실패율 계산: (스테이지 번호, 실패율) 쌍 생성
    let mut fail_rates: Vec<(usize, f64)> = Vec::with_capacity(n);
    let mut reached = stages.len(); // 현재 스테이지에 도달한 누적 인원

    for i in 1..=n {
        let rate = if reached > 0 {
            // 실패율 = 해당 스테이지에 멈춘 수 / 도달한 수
            count[i] as f64 / reached as f64
        } else {
            // 도달한 사람이 없으면 실패율은 0
            0.0
        };
        fail_rates.push((i, rate));
        // 다음 스테이지 도달 인원 차감
        reached -= count[i];
    }

    // 실패율 내림차순 정렬 (동률 시 스테이지 번호 오름차순)
    fail_rates.sort_by(|a, b| {
        b.1.partial_cmp(&a.1)
            .unwrap_or(std::cmp::Ordering::Equal)
            .then(a.0.cmp(&b.0))
    });

    // 스테이지 번호만 추출
    fail_rates.into_iter().map(|(stage, _)| stage).collect()
}
```

### Go

```go
import "sort"

func solution(N int, stages []int) []int {
    // 각 스테이지에 멈춰있는 플레이어 수를 카운팅
    count := make([]int, N+2)
    for _, s := range stages {
        count[s]++
    }

    // 실패율 계산을 위한 구조체 슬라이스
    type stageInfo struct {
        num  int
        rate float64
    }
    failRates := make([]stageInfo, N)
    reached := len(stages) // 현재 스테이지에 도달한 누적 인원

    for i := 1; i <= N; i++ {
        rate := 0.0
        if reached > 0 {
            // 실패율 = 해당 스테이지에 멈춘 수 / 도달한 수
            rate = float64(count[i]) / float64(reached)
        }
        failRates[i-1] = stageInfo{num: i, rate: rate}
        // 다음 스테이지 도달 인원 차감
        reached -= count[i]
    }

    // 실패율 내림차순 정렬 (동률 시 스테이지 번호 오름차순)
    sort.SliceStable(failRates, func(a, b int) bool {
        if failRates[a].rate != failRates[b].rate {
            return failRates[a].rate > failRates[b].rate
        }
        return failRates[a].num < failRates[b].num
    })

    // 스테이지 번호만 추출
    result := make([]int, N)
    for i, info := range failRates {
        result[i] = info.num
    }
    return result
}
```

## 7. 단계별 코드 해설 (배열 풀이 기준)

**단계 1 — 카운팅 배열 생성.** stages 배열을 한 번 순회하면서 각 스테이지 번호에 몇 명이 멈춰있는지 센다. 배열 크기를 N+2로 잡는 이유는 stages에 N+1 값(모든 스테이지를 클리어한 사람)이 들어올 수 있기 때문이다.

**단계 2 — 누적 차감으로 실패율 계산.** `reached`를 전체 플레이어 수로 초기화한 뒤, 스테이지 1부터 N까지 순서대로 실패율을 구한다. 각 스테이지를 처리한 후 reached에서 `count[i]`를 빼면, 다음 스테이지에 도달한 사람 수가 자동으로 나온다. 이것이 왜 정확한지 직관적으로 설명하면, 스테이지 i에 "도달한" 사람은 스테이지 i-1까지 모두 클리어한 사람이다. 스테이지 1에서 멈춘 1명은 스테이지 2에 도달하지 못했으므로, 8 - 1 = 7명이 스테이지 2에 도달한다.

**단계 3 — 정렬.** 계산된 (스테이지 번호, 실패율) 쌍을 실패율 내림차순으로 정렬한다. 동률인 경우 번호가 작은 스테이지가 먼저 오도록 2차 정렬 기준을 적용한다.

## 8. 엣지 케이스 분석

| 관점 | 구체적 케이스 | 처리 방법 |
|---|---|---|
| **도달 인원 0** | 앞 스테이지에서 모든 플레이어가 멈춰 이후 스테이지에 아무도 도달하지 않는 경우 | `reached == 0` (또는 `challenger[i] == 0`) 검사로 0 나누기 방지 |
| **모두 클리어** | stages의 모든 원소가 N+1 (예: N=4, stages=[5,5,5]) | 모든 스테이지의 count가 0이므로 실패율 전부 0, 번호 오름차순 반환 |
| **단일 스테이지** | N=1 | 스테이지 1개의 실패율만 계산하여 [1] 반환 |
| **모두 같은 스테이지** | stages=[3,3,3,3,3], N=5 | 스테이지 1,2는 count=0이지만 reached > 0이므로 실패율 0. 스테이지 3은 실패율 1.0. 스테이지 4,5는 reached=0이므로 실패율 0 |
| **동률 정렬** | 여러 스테이지의 실패율이 동일 | 비교 함수에서 번호 오름차순 2차 정렬로 처리 |
| **N+1 값 처리** | stages에 N+1이 포함 | 카운팅 배열 크기 N+2로 잡아 인덱스 초과 방지. N+1은 실패율 계산 대상이 아니지만 reached에서 자연스럽게 차감됨 |

## 9. 복잡도 분석

| 풀이 | 시간 복잡도 | 공간 복잡도 | 비고 |
|---|---|---|---|
| 카운팅 + 정렬 (배열/HashMap) | O(M + N log N) | O(N) | M = stages 길이, 정렬은 N개 원소 대상. 최적 해법 |
| 직접 카운팅 (대안) | O(N × M) | O(N) | 구현 직관적이나 비효율 |

시간 복잡도를 풀어 설명하면, 카운팅에 O(M), 실패율 계산에 O(N), 정렬에 O(N log N)이 들어 전체는 O(M + N log N)이다. N ≤ 500, M ≤ 200,000이므로 매우 여유로운 시간 안에 동작한다.

## 10. 대안 풀이 — 매 스테이지마다 직접 카운팅

카운팅 배열을 미리 만들지 않고, 각 스테이지마다 stages 배열을 직접 순회하여 "도달한 수"와 "멈춘 수"를 세는 방법이다. 구현이 더 직관적이지만 시간 복잡도가 O(N × M)으로 늘어난다.

### JavaScript (대안)

```javascript
function solution(N, stages) {
    const failRates = [];

    for (let i = 1; i <= N; i++) {
        // 스테이지 i에 도달한 사람 수 (i 이상에 있는 모든 플레이어)
        const reached = stages.filter(s => s >= i).length;
        // 스테이지 i에서 멈춘 사람 수 (정확히 i에 있는 플레이어)
        const stuck = stages.filter(s => s === i).length;
        // 실패율 계산 (도달 0이면 실패율 0)
        const rate = reached === 0 ? 0 : stuck / reached;
        failRates.push([i, rate]);
    }

    // 실패율 내림차순, 동률 시 번호 오름차순
    failRates.sort((a, b) => b[1] - a[1] || a[0] - b[0]);
    return failRates.map(([stage]) => stage);
}
```

### Java (대안)

```java
import java.util.*;

class Solution {
    public int[] solution(int N, int[] stages) {
        double[][] failRates = new double[N][2];

        for (int i = 1; i <= N; i++) {
            int reached = 0; // 스테이지 i에 도달한 사람 수
            int stuck = 0;   // 스테이지 i에서 멈춘 사람 수

            for (int s : stages) {
                if (s >= i) reached++;
                if (s == i) stuck++;
            }

            failRates[i - 1][0] = i;
            failRates[i - 1][1] = reached == 0 ? 0 : (double) stuck / reached;
        }

        Arrays.sort(failRates, (a, b) -> {
            if (Double.compare(b[1], a[1]) != 0) return Double.compare(b[1], a[1]);
            return Double.compare(a[0], b[0]);
        });

        int[] result = new int[N];
        for (int i = 0; i < N; i++) result[i] = (int) failRates[i][0];
        return result;
    }
}
```

두 풀이의 차이를 비유하면, **최적 풀이**는 출석부를 한 번 돌면서 반별 인원을 미리 기록해두는 방식이고, **대안 풀이**는 매 반마다 전교생에게 "너 이 반이야?"라고 물어보는 방식이다.

## 11. 핵심 포인트 정리

**알고리즘 원리**: 카운팅 배열 + 누적 차감은 "각 단계에 도달한 인원"을 O(1)로 구하는 표준 테크닉이다. 매번 전체 배열을 순회하는 O(M)을 피할 수 있다.

**Java 타입 승격**: `int / double`은 자동으로 double 나눗셈이 된다. 내 풀이에서 `total`을 double로 선언한 것은 이 규칙을 활용한 깔끔한 접근이다.

**0 나누기 방어**: 도달한 사람이 0명인 경우 실패율을 0으로 정의해야 한다. `challenger[i] == 0` 검사(내 풀이) 또는 `reached > 0` 검사(배열 풀이) 중 하나로 방어할 수 있다.

**커스텀 정렬**: 다중 기준 정렬에서 1순위가 같을 때 2순위로 넘어가는 패턴은 코딩 테스트에서 자주 등장한다. `a.equals(b) ? 2차비교 : 1차비교` 패턴(Stream) 또는 `compare(b,a) != 0 ? 그 값 : compare(a의2차, b의2차)` 패턴(Arrays.sort)을 기억해두면 유용하다.

**배열 크기 N+2**: stages에 N+1 값이 들어올 수 있으므로, 카운팅 배열 크기를 N+2로 잡아야 인덱스 초과를 방지한다. 이것은 "문제의 값 범위를 정확히 읽어야 한다"는 교훈과 직결된다.
